# 판례 AI 학습 서비스 구현 로드맵

본 문서는 `PRD.md` v4.0을 기준으로 서비스를 단계별로 구현하기 위한 실행 로드맵이다. 목표는 공식 법률 데이터 기반 RAG 구조를 먼저 안정화하고, 그 위에 IRAC 분석, 판례 검증, 비교 분석, 구독 기능을 순차적으로 얹는 것이다.

---

## 전체 개발 전략

### 핵심 방향

1. 먼저 모듈식 백엔드 구조를 만든다.
2. 외부 공식 API 연동과 RAG 파이프라인을 가장 먼저 안정화한다.
3. Gemini는 RAG 근거가 충분할 때만 호출한다.
4. 모든 AI 응답은 JSON schema와 evidence 기반 검증을 통과해야 한다.
5. 프론트엔드는 전문가 모드 MVP를 먼저 만들고, 일반인 모드는 후순위로 확장한다.

### 우선순위

| 우선순위 | 영역 | 이유 |
| :--- | :--- | :--- |
| 1 | RAG 검색/근거 수집 | 정확도와 환각 방지의 기반 |
| 2 | Gemini 호출 제한/캐시 | 비용 절감과 중복 호출 방지 |
| 3 | IRAC 분석 | MVP 핵심 가치 |
| 4 | 판례 검증 | 공식 데이터 기반 신뢰성 확보 |
| 5 | 프론트엔드 분석 UI | 사용자 검증 가능성 확보 |
| 6 | 구독/저장/비교 | 수익화와 고급 기능 |
